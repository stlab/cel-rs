# Begin: npm + esbuild Spectrum Bundle Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the Spectrum 2 theme-token regression (confirmed real, root-caused, and
already proven fixable in a standalone spike) by replacing `begin`'s fragmented
Spectrum asset vendoring (5 separate script tags, 2 of which are provably broken) with
one esbuild-compiled bundle covering elements, Spectrum 2 theme tokens, and the
zoom-control icons together.

**Architecture:** `begin/package.json` (npm, exact-pinned devDependencies) +
`begin/js/spectrum-entry.js` (one file, five side-effect imports) + esbuild
(`--bundle --format=esm --loader:.css=css`) produce a single `begin/assets/swc.js`,
regenerated via `cargo xtask build-js` (a manual, occasional step whose *output* is
committed — mirrors `cargo xtask fetch-assets`, so most contributors never need Node
installed). `app.rs` collapses from three Spectrum-related script tags to one.
Verified against a real, working standalone spike before this plan was written
(esbuild 0.24.2, Node v26.4.0): the produced bundle's `sp-theme` adopted 3 stylesheets
(was 0/unreachable) and `sp-action-button`'s computed font was the real Spectrum stack
(was `"Times New Roman"`).

**Tech Stack:** Rust (`xtask`), Node.js/npm (occasional, build-only dependency),
esbuild 0.24.2, Spectrum Web Components 1.12.2, Dioxus 0.7, `verifying-begin-ui` skill.

## Global Constraints

- All npm devDependency versions are exact-pinned (no `^`/`~` ranges) — matches the
  precision-pinning style already used throughout `versions.toml`.
- `begin/node_modules/` must be gitignored. `begin/package.json`,
  `begin/package-lock.json`, and `begin/js/spectrum-entry.js` must be committed.
  `begin/assets/swc.js` (the build *output*) is also committed, same as every other
  vendored asset in this repo — contributors who never touch the Spectrum integration
  must never need Node installed.
- `cargo xtask build-js` must work cross-platform: on Windows, `npm` must be invoked
  as `npm.cmd` (Rust's `Command::new("npm")` fails to find the shim on Windows without
  the extension); on other platforms, `npm`.
- **Never `taskkill //IM msedge.exe` (or any kill-by-process-name) during
  verification** — always capture and kill the specific PID this session launched, and
  always pass `--user-data-dir=/tmp/begin-ui-profile` to headless Edge invocations, per
  `.claude/skills/verifying-begin-ui/SKILL.md`'s current guidance. This is a hard
  requirement, not a style preference — a blanket kill takes down the human's own
  browser windows.
- This fix cannot be verified by `cargo build`/`cargo clippy` alone — the regression
  and two prior fix attempts were all invisible to both. The proof is the live
  DevTools Protocol query from the spec's Verification section, run inside the real
  `begin` app (not just the standalone spike), via the `verifying-begin-ui` skill.
- Before opening a PR: `cargo fmt --all`, a zero-warning `cargo build --workspace` and
  `cargo test --workspace`, and all three clippy invocations (`cargo clippy
  --workspace --exclude begin --all-targets -- -D warnings`, `cargo clippy -p begin
  --no-default-features --all-targets -- -D warnings`, `cargo clippy -p begin
  --all-targets -- -D warnings`) must pass, per `CLAUDE.md`.

- [ ] **Step 5: Add the `build-js` subcommand to `xtask/src/main.rs`**

Add this function (after `fetch_assets`):

```rust
/// Runs `npm ci` then `npm run build` in `begin/` to regenerate the vendored
/// Spectrum Web Components bundle (`begin/assets/swc.js`) from `begin/package.json`
/// and `begin/js/spectrum-entry.js`.
///
/// # Errors
/// Returns `Err` if `npm` is not on `PATH`, or if either command exits non-zero.
fn build_js() -> Result<(), Box<dyn std::error::Error>> {
    let root = project_root();
    let begin_dir = root.join("begin");
    let npm = if cfg!(windows) { "npm.cmd" } else { "npm" };

    let steps: [&[&str]; 2] = [&["ci"], &["run", "build"]];
    for args in steps {
        println!(
            "Running `npm {}` in {} ...",
            args.join(" "),
            begin_dir.display()
        );
        let status = std::process::Command::new(npm)
            .args(args)
            .current_dir(&begin_dir)
            .status()?;
        if !status.success() {
            return Err(format!("npm {} failed with {status}", args.join(" ")).into());
        }
    }

    Ok(())
}
```

Replace the `main` function's dispatch:

```rust
fn main() {
    match std::env::args().nth(1).as_deref() {
        Some("fetch-assets") => {
            if let Err(e) = fetch_assets() {
                eprintln!("error: {e}");
                std::process::exit(1);
            }
        }
        Some("build-js") => {
            if let Err(e) = build_js() {
                eprintln!("error: {e}");
                std::process::exit(1);
            }
        }
        _ => {
            eprintln!("Usage: cargo xtask <fetch-assets|build-js>");
            std::process::exit(1);
        }
    }
}
```
